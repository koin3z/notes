## 概要

ISO/IEC 20000-1 は、Service Management System（SMS）を確立、実施、維持し、継続的に改善するための
要求事項を定める国際規格である。現行版は ISO/IEC 20000-1:2018。

サービスの計画、設計、移行、提供、改善を一貫して管理し、サービス要求を満たして価値を提供することを
目的とする。情報セキュリティ専用の規格ではないが、安定したセキュリティ運用と密接に関係する。

## 主な領域

- 組織の状況、リーダーシップ、計画、支援、評価、改善
- サービスポートフォリオとサービス要求
- 関係・合意、サービスレベル、供給者管理
- 需要と容量、可用性、サービス継続性
- 情報セキュリティ管理
- インシデント、サービス要求、問題管理
- 構成管理、資産管理、変更管理、リリース・展開管理

## セキュリティ業務との接点

| サービス管理     | セキュリティでの利用例                   |
| ---------------- | ---------------------------------------- |
| 構成管理         | 影響を受ける資産、所有者、依存関係の特定 |
| 変更管理         | セキュリティレビュー、承認、ロールバック |
| インシデント管理 | 検知、優先度、連絡、復旧、記録           |
| 問題管理         | 根本原因分析、再発防止、既知エラー管理   |
| 可用性・継続性   | RTO/RPO、冗長化、バックアップ、復旧演習  |
| 供給者管理       | SLA、セキュリティ条項、委託先の実績監視  |

脆弱性対応でも、修正パッチを当てるだけでなく、変更影響、停止調整、例外、復旧、問題管理まで連携する。

## ISO/IEC 27001との違い

- [[memos/iso-iec-27001|ISO/IEC 27001]]: 情報セキュリティリスクを管理する ISMS の要求事項
- ISO/IEC 20000-1: IT サービスの品質とライフサイクルを管理する SMS の要求事項

両規格は共通するマネジメントシステム要素を持ち、統合運用できる。どちらかの認証が他方の認証を自動的に
意味するわけではない。

## 認証確認のポイント

- 適用される規格の版
- 認証された組織、サービス、所在地、供給者の範囲
- 自分が利用するサービスが適用範囲に含まれるか
- SLA、可用性、継続性、インシデント対応の実績
- セキュリティ要求が SMS と ISMS のどちらで管理されるか
