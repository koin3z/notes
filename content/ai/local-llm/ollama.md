## Ollama とは

ローカル環境（自分の PC やオンプレミスサーバー）で大規模言語モデルを簡単にダウンロード・実行・管理するためのオープンソースプラットフォーム。

直感的な理解のための例として「LLM のための Docker」が最もわかりやすい。コンテナで Web サーバーを立ち上げる際に `docker run nginx` と打つように、AI を立ち上げる際は `ollama run qwen3:14b` とコマンドを打つだけ。複雑な環境構築なしに AI がローカル PC 上で動き出す。

単なる「チャット用ツール」ではなく、LLM をシステムの一部として組み込んで開発や自動化に利用するための**軽量なインフラ基盤**である、というのが Ollama。

（参考元：[Ollama 公式サイト](https://ollama.com/) / [GitHub リポジトリ](https://github.com/ollama/ollama)）

## 全体像：Ollama の役割

これまでローカルで AI を動かすには、Python の仮想環境の構築・CUDA（GPU 用ドライバ）のバージョン合わせ・モデルファイルの依存関係の解決など、インフラ構築に近い複雑なハードルがあった。Ollama はこれらの処理をすべて C++（llama.cpp ベース）などで隠蔽し、1 つの実行ファイルにまとめて提供する。

## 詳細：なぜ Ollama がスタンダードになったのか

Ollama が現在のローカル LLM 運用において主流となっている理由は、主に以下の 3 つの技術的な特徴による。

### 1. 「Modelfile」による構成管理

Docker における `Dockerfile` のように、Ollama には `Modelfile` という構成ファイルの仕組みがある。

ベースとなるモデル（Qwen や Llama など）を指定し、そこにシステムプロンプト（AI の基本動作）やパラメーター（回答のランダム性など）を記述してビルドすることで、自分専用にカスタマイズされた AI エージェントの環境をコードとして管理（IaC 化）できる。

### 2. バックグラウンドでの API サーバー機能

Ollama はターミナルで対話するだけでなく、起動するとバックグラウンドで **REST API サーバー**（デフォルトでは `http://localhost:11434`）として常駐する。

他のアプリケーションから API 経由でローカル AI を呼び出すことが可能になる。例えば、Neovim のプラグインからコード補完のために呼び出したり、Python で作った自作スクリプトから推論エンジンとして利用したりすることが非常に簡単になる。

### 3. ハードウェアへの自動最適化とメモリ管理

Ollama は、実行される PC のハードウェア構成を自動で検知する。モデルを動かすために必要な VRAM（ビデオメモリ）が不足している場合でも、自動的に CPU 側のメインメモリ（RAM）に処理をオフロード（分割）して、システムをクラッシュさせることなく実行を継続する高度なメモリ管理機構を持っている。
