プロセスを生成する方法


|方法|意味|
|---|---|
|`fork()`|現在のプロセスをコピーして子プロセスを作る|
|`vfork()`|`fork()` に似ているが、子がすぐ `exec()` する前提の特殊な方法|
|`clone()`|Linux固有の低レベルAPI。プロセスやスレッド生成の土台|
|`clone3()`|`clone()` の新しい拡張版|
|`posix_spawn()`|`fork()` + `exec()` に近い処理をまとめて行う|
|`system()`|シェル経由でコマンドを実行する|
|`popen()`|コマンドを実行し、その入出力をパイプで扱う|

## fork とは
`fork()` は、Linux/Unix系OSで **現在動いているプロセスをもとに、子プロセスを作る仕組み**。

イメージ：
```text
親プロセス
	|  
	+-- 子プロセス
```


`fork()` を呼ぶと、親プロセスと子プロセスの **2つのプロセス** に分かれる。

---

## 子プロセスはどこから動くか

子プロセスは `main()` の最初から動くわけではない。

`fork()` の直後から、親プロセスと子プロセスの両方が実行を続ける。

```
printf("A\n");
fork();
printf("B\n");
```

この場合：

```
A
B
B
```

となる。

理由：

- `A` は `fork()` の前なので、1回だけ表示される
- `B` は `fork()` の後なので、親と子の両方が実行する
- そのため `B` は2回表示される

---

## fork の戻り値

`fork()` の戻り値を使うと、親プロセスと子プロセスを区別できる。

```
pid_t pid = fork();
if (pid == 0) { 
	// 子プロセス
} else {
    // 親プロセス
}
```

ポイント：

```
子プロセスでは pid == 0
親プロセスでは pid == 子プロセスのPID
```

---

## fork が複数ある場合

`fork()` が1回実行されるたびに、そこを通るプロセス数は基本的に2倍になる。

```
printf("A\n");
fork();
fork();
printf("B\n");
```

流れ：

```
最初: 1プロセス

1回目の fork()
1 → 2プロセス

2回目の fork()
2 → 4プロセス
```

したがって：

```
A は 1回
B は 4回
```

---

## 覚え方

```
fork = 分身を作る
exec = 中身を別プログラムに入れ替える
fork + exec = 新しいプログラムを別プロセスで実行する
```

---

## 注意：exec はプロセス生成ではない

`exec()` は新しいプロセスを作るものではない。

今のプロセスの中身を、別のプログラムに置き換える。

よくある流れ：

```
fork() で子プロセスを作る
↓
子プロセス側で exec() する
↓
別のプログラムとして動く
```

シェルで `ls` などを実行するときも、概念的にはこの流れになる。

```
shell が fork()
子プロセスが exec("ls")
親の shell は待つ
```

---

## 重要ポイントまとめ

- `fork()` は子プロセスを作る
- 子プロセスは `main()` の最初からではなく、`fork()` の直後から動く
- `fork()` の後の処理は、親と子の両方が実行する
- `fork()` が増えると、プロセス数は2倍ずつ増える
- `exec()` はプロセス生成ではなく、プロセスの中身を置き換える処理
